Our Tasks & Dispatch functionality allows for tasks to be sent out to users' devices out in the field.

Tasks can either be to Fill Out a Form, a set of Instructions, Read a Document, or multiple combinations of the three.

Dispatching isn't limited to a single user but can also be sent out to all users or a group of users with options for All to Complete, First Available Person, First User to Claim or Unassigned Task User.

How you create tasks is entirely up to you, whether it's a one-off task or recurring that can be swiftly set up and dispatched especially when using a task template.

Typically, task creation and dispatch are achieved using the web portal but our Process Step field and Tasking API can accomplish similar outcomes via in-app solutions or external system communications.

Web Portal

When logged into the web portal and navigating to the Tasks & Dispatch menu area, the following areas will be available to you.

Each area provides options to view, create and dispatch tasks respectively. We suggest jumping into your account, familiarising yourself with these areas by sending out a few tasks and getting a feel for receiving tasks on your device.

In-App

When a task is received by a user, a push notification is sent to the device that can launch the app displaying the task.

In-app, Task Notifications display at the bottom of the app's start screen.

Upon selecting, the user's Task List will display allowing task selection and completion.

  1. Task name

  2. Expected start and due dates

  3. Task status

  4. Task search text box

  5. Task status filter

Once a task is claimed, if sent to numerous users and not all to complete, the task status changes to assigned and is removed from all other users' task list.
